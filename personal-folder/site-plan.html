Overview

Purpose

The purpose of this website is to create a professional online portfolio to showcase my photography work. It will serve as a central hub where I can display high-quality images taken with my Canon camera, allowing me to share my hobby with friends and family, and demonstrate my creative skills to potential employers.

Audience

The primary audience includes photography enthusiasts, friends, family, and potential clients looking for a photographer. They will access the site from both desktop computers and mobile devices, so the site must be fully responsive. They are looking for a clean, visual-heavy experience where they can easily filter photos by category (e.g., Nature vs. Urban).

Dynamic elements

I will use JavaScript to dynamically generate the photo gallery on the secondary page.

1. ES Modules: I will store my photo data in a separate file (photos.js) and import it into my main script.
2. Objects & Arrays: My data will be an array of objects. Each object will represent a photo with properties like id, category, src, and description.
3. Array Methods: I will use .map() to generate the HTML cards for the gallery and .filter() to let users sort images by category buttons.

Paragraph Font: "Open Sans", sans-serif

Normal paragraph example

Photography is the art of capturing light. This website is designed to let the images speak for themselves. By using a clean font like Open Sans, we ensure that the text supports the visuals rather than distracting from them. Each gallery item will feature a short description explaining the setting and equipment used.

Colored paragraph example

This is how text will look when placed over our primary color background. It is important to ensure high contrast for accessibility, making sure all visitors can read the descriptions of the photos easily.

Content

Home page

The home page will be simple and inviting. It will feature:
1. A "Hero" image (a large, high-quality banner photo) that immediately sets the tone.
2. A brief introduction about me and my passion for photography.
3. A "Call to Action" button that says "View Gallery" to direct users to the main content.

Gallery Page (Page 2)

This is the main child page. It will contain:
1. Filter buttons at the top (e.g., "Nature", "Urban", "Portraits", "All").
2. A responsive grid of image cards.
3. Each card will have the photo, a title, and a small description.
4. This content will be loaded via JavaScript modules.
